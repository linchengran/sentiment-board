# 舆情监控看板

一个面向个人使用的多来源舆情监控网页应用。用户可以创建监控任务，定时采集视频评论、热门话题、RSS 订阅或自定义文本，系统会自动进行情感分析、趋势聚合、负面占比统计、关键词提取和舆情拐点检测。

## 项目亮点

- 多来源采集：支持 B站视频评论、B站关键词视频评论、B站热门视频、微博热搜、知乎热榜、RSS 和自定义文本。
- 自动化监控：任务支持启用/暂停、手动执行、按间隔自动执行。
- 情感趋势分析：按时间粒度聚合平均情感分和负面占比，识别口碑变化。
- 舆情拐点检测：基于情感分变化和负面占比变化标记异常波动。
- 可视化看板：Flask 提供 API，原生 HTML/CSS/JS 绘制趋势图、任务列表、关键词和明细表。
- 可演示数据：内置一键演示数据，避免作品展示时依赖第三方平台接口稳定性。

## 技术栈

- 后端：Flask、Requests、Pandas
- 前端：HTML、CSS、原生 JavaScript、SVG
- 数据：CSV 持久化历史文本，JSON 持久化监控任务
- NLP：优先复用项目根目录 `analyze_bilibili.py` 的情感模型；模型不可用时降级为词典规则打分

## 各平台使用说明

### 通用流程

1. 点击页面右上角”新建任务”。
2. 选择数据源类型，填写监控目标（见各平台说明）。
3. 设置采集间隔（分钟）和最大页数/条数。
4. 点击”创建”，再点击任务卡片上的”执行”立即采集一次。
5. 开启”运行中”开关，后端会按间隔自动重复采集。

---

### B站视频评论

| 字段 | 说明 |
|------|------|
| 监控目标 | BV 号（如 `BV1xx411c7mD`）或完整视频链接 |
| Cookie | 可选；不填也能抓到大部分公开评论 |
| 最大页数 | 每页约 20 条评论，3 页 = 约 60 条 |

**步骤**：打开目标视频页面，复制地址栏中的 BV 号或完整 URL，粘贴到”监控目标”框。

**注意**：B站有 WBI 反爬签名机制，若频繁请求会触发风控（返回 -352 错误），此时适当延长采集间隔（建议 ≥ 10 分钟）。

---

### B站话题 / 关键词

| 字段 | 说明 |
|------|------|
| 监控目标 | 关键词，如 `某电影名`、`某品牌` |
| 最多视频数 | 搜索结果中取前 N 个视频，再各抓评论 |

**步骤**：填入想监控的话题词，系统会自动搜索相关视频并聚合其评论。适合跟踪某品牌、事件或产品的整体舆论。

---

### B站热门视频

| 字段 | 说明 |
|------|------|
| 监控目标 | 可选；留空则抓所有热门，填关键词则只取标题含该词的视频 |

**步骤**：直接创建任务，无需填写目标（或填入过滤词）。适合监控平台整体舆论氛围。

---

### 微博热搜

| 字段 | 说明 |
|------|------|
| 监控目标 | 可选过滤关键词；留空则抓取全部热搜词条 |
| Cookie | **强烈建议填写**，否则接口可能返回空数据 |

**获取 Cookie 步骤**：
1. 浏览器登录 [weibo.com](https://weibo.com)。
2. 按 F12 → Network → 刷新页面 → 找任意 `weibo.com` 请求。
3. 在请求头 `Cookie:` 字段中，复制完整 cookie 字符串，粘贴到任务的 Cookie 框。

**说明**：采集的是热搜词条文本（词 + 附注），不是具体帖子内容；适合监控热点话题的情感倾向。

---

### 知乎热榜

| 字段 | 说明 |
|------|------|
| 监控目标 | 可选过滤关键词；留空则抓取全部热榜条目 |
| Cookie | **建议填写**，不填时部分接口可能受限 |

**获取 Cookie 步骤**：
1. 浏览器登录 [zhihu.com](https://www.zhihu.com)。
2. 按 F12 → Network → 访问 [知乎热榜](https://www.zhihu.com/hot) → 找到 `hot-lists` 请求。
3. 复制请求头中的 `Cookie:` 值，粘贴到任务的 Cookie 框。

**说明**：采集热榜问题标题 + 摘要文本，并附上热度指标；适合追踪公众关注焦点的情绪变化。

---

### RSS / 网页订阅

| 字段 | 说明 |
|------|------|
| 监控目标 | RSS feed 的完整 URL，必须以 `http://` 或 `https://` 开头 |

**常用 RSS 示例**：

```text
# 36氪（科技资讯）
https://36kr.com/feed

# 少数派
https://sspai.com/feed

# 虎嗅
https://www.huxiu.com/rss/0.xml

# 微信公众号 / 博客：可通过 RSSHub 转换，例如
https://rsshub.app/bilibili/user/video/你的UID
```

**说明**：系统解析 RSS/Atom 格式，提取文章标题和摘要进行情感分析；不支持需要登录的付费内容。

---

### 自定义文本

| 字段 | 说明 |
|------|------|
| 监控目标 | 每行一条文本，直接粘贴评论或内容 |

**适用场景**：
- 从微信群、微信公众号、小红书、抖音评论区手动复制内容粘贴分析。
- 从其他平台导出 CSV 后，把评论列复制进来。
- 一次性分析某批文字，不需要重复采集。

**示例**：
```text
这款产品外观很不错，用了一周没问题
价格偏高，性价比一般
客服响应很快，体验还不错
质量堪忧，买了就后悔
```

## 使用建议

1. 先点击页面右上角”载入演示数据”，确认趋势图、拐点和明细效果正常。
2. 对于 B 站，先用”视频评论”或”话题/关键词”模式测试，确保能抓到数据再设定自动间隔。
3. 微博和知乎建议提供 Cookie，否则接口容易返回空数据。
4. RSS 订阅稳定性最高，无需 Cookie，适合持续监控媒体报道情绪。
5. 若要快速展示多平台效果，可先用”自定义文本”导入各平台样本内容。
6. 长期监控时将任务保持”运行中”，后端按设置的间隔自动采集；单次分析可选”执行一次”模式，跑完自动停用。
